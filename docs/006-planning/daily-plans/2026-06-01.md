## 3. 오늘 맞춰야 할 기준

| 구분 | 내용 | 담당 |
|---|---|---|
| API | 모든 API 응답은 `code/status/message/data` 공통 응답으로 감싼다 | 전체 백엔드, 황희수 |
| DB | 상태값/권한/department 관련 컬럼은 도메인 담당자끼리 충돌 여부 확인 | 백엔드 전체 |
| DB | 최신 migration 보강사항은 `categories`, `department_category_mappings`, `ticket_status_logs`, `user_points`, `manuals.status`, RAG 출처 FK/청크 출처 구조를 기준으로 확인한다 | 백엔드 전체 |
| 화면 | 질문과 요청 진입은 분리하고, 챗봇 실패 시 요청 티켓 전환 흐름을 유지 | 황희수, 김진혁 |
| 권한 | `USER`, `TEAM_ADMIN`, `SYSTEM_ADMIN` 기준으로 API 접근 범위를 나눈다 | 이슬이, 김가영, 김진혁 |
