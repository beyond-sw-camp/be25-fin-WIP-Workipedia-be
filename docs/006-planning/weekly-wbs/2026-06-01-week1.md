## 2.1 황희수 FE 작업 기준

황희수는 FE skeleton 구현 전에 Figma Make 화면이 최신 기획 문서와 맞는지 먼저 확인한다.

확인 기준:

- 메인 진입이 `질문`과 `요청`으로 분리되어 있는가
- 질문 -> 챗봇 -> 매뉴얼 + 워키 출처 답변 흐름이 보이는가
- 답변 실패/불충분 시 요청 티켓 발행 액션이 있는가
- 요청 -> 티켓 발행 -> 자동 배정/공통 접수 큐 흐름이 표현되는가
- `TEAM_ADMIN`이 자기 팀 티켓을 확인하고 팀원에게 배정하는 화면이 있는가
- `SYSTEM_ADMIN`이 공통 접수 큐를 확인하는 화면이 있는가
- 처리 완료 티켓의 지식화 후보/검수 화면이 있는가
- 포인트/뱃지/ESG 카드는 MVP 수준으로라도 노출되는가

수정 기준:

- 큰 화면 구조나 시연 동선 변경은 Figma Make에서 먼저 맞춘다.
- 버튼명, 문구, 상태 뱃지, API 연동에 따른 세부 UI는 FE 코드에서 맞춘다.
- Figma와 문서가 충돌하면 `docs/001-reference/service-flow.md`와 `docs/001-reference/project-proposal.md`를 기준으로 한다.

## 4. 매일 종료 시 작성할 daily report

매일 작업 종료 후 담당자는 `docs/006-planning/daily-reports/`에 본인 daily report를 남긴다.
팀 합의가 필요한 논의사항은 필요할 때만 `docs/006-planning/daily-discussions/`에 정리한다.

파일명 규칙:

```text
YYYY-MM-DD/{member}.md
```

예시:

```text
docs/006-planning/daily-reports/2026-06-02/kim-jinhyeok.md
```

작성 기준:

- 오늘 완료한 것
- 오늘 못 끝낸 것
- 내일 오전에 결정해야 하는 것
- API/DB/화면 영향이 있는 변경
- 다른 담당자에게 필요한 요청
