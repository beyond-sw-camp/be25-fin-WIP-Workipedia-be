본 문서는 2026-05-29 금요일 기준으로 확정하는 다음 주 실행 WBS다.
주간 목표는 큰 기능 완성이 아니라 **각 담당 도메인의 skeleton을 만들고, 프론트/백엔드가 같은 API 계약으로 대화할 수 있는 상태**를 만드는 것이다.

## 1. 주간 목표

| 목표 | 완료 기준 |
|---|---|
| BE skeleton 생성 | Auth, 챗봇 세션, 티켓, 워키, 관리자/포인트의 기본 Controller/Service/Repository/Entity 구조가 잡힘 |
| FE skeleton 생성 | 라우팅, 공통 레이아웃, 로그인, 챗봇, 워키, 티켓, 관리자 mock 화면이 열림 |
| API 계약 1차 정리 | 프론트가 화면을 만들 수 있는 request/response 예시가 `docs/004-api/api-contract.md`에 반영됨 |
| 핵심 흐름 mock 확인 | 로그인 -> 챗봇 mock 질문 -> 티켓 mock 생성 -> 관리자 mock 조회 흐름을 설명할 수 있음 |

## 4. 매일 종료 시 작성할 다음날 논의사항

```text

```

```text

```

작성 기준:

- 오늘 완료한 것
- 오늘 못 끝낸 것
- 내일 오전에 결정해야 하는 것
- API/DB/화면 영향이 있는 변경
- 다른 담당자에게 필요한 요청
