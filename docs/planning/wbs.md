## 운영 원칙

- 각 팀원은 본인 `member-wbs/{이름}.md`를 직접 관리한다.
- PRD 본인 담당 섹션, API 계약, DB 스키마는 개발하면서 직접 채운다.
- 마일스톤 목표 기준으로 진행 상황을 파악한다. 일별 계획은 팀원 자율에 맡긴다.

| Elasticsearch | 민정기 | 벡터 스토어, 매뉴얼/워키 청크 인덱싱, kNN 검색 |

| Flash Chat | 김진혁 + 민정기 | 김진혁: WebSocket/STOMP, Redis TTL, 관리자 설정. 민정기: BE 완료 후 실시간 채팅 UI |
| 사진 첨부 | 김진혁 + 황희수 | 김진혁: R2/S3/MinIO 추상화와 presigned API. 황희수: 파일/카메라 직접 업로드 UI |
| 지식화 | 김진혁 + 김가영 | 지식화 생성·동기화(김진혁), TEAM_ADMIN 승인(김가영) |
| 관리자 대시보드·부서 | 김가영 | TEAM_ADMIN/SYSTEM_ADMIN 큐, admin_logs, 부서 CRUD와 운영 지표 |
| 포인트/ESG | 이슬이 | 포인트 이력, ESG 등급, ESG 지표 카드 |
| 프론트엔드 | 황희수 | 전체 화면, API 연동, 발표 시연 UX |
| 모바일 대응 | 민정기 | BE 완료 후 반응형 UI, PWA/Capacitor 래핑 검토, 현장 사진 첨부 흐름 |
| CDN 스크립트 주입 | 민정기 | 챗봇 컴포넌트 빌드, 스크립트 주입 가이드, CORS 허용 도메인 연계 |
