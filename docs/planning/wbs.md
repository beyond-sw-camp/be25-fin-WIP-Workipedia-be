## 운영 원칙

- 각 팀원은 본인 `member-wbs/{이름}.md`를 직접 관리한다.
- PRD 본인 담당 섹션, API 계약, DB 스키마는 개발하면서 직접 채운다.
- 마일스톤 목표 기준으로 진행 상황을 파악한다. 일별 계획은 팀원 자율에 맡긴다.

## 역할 경계

| 영역 | 담당 | 경계 |
|---|---|---|
| Auth | 이슬이 | 회원가입, 로그인, JWT, USER/TEAM_ADMIN/SYSTEM_ADMIN 권한, 내 정보 |
| 챗봇 | 이슬이 + 김진혁 | 이슬이: 세션/메시지/저장. 김진혁: 응답 생성/RAG/전환 |
| 워키 게시판 | 민정기 | 질문, 답변, 채택, 반응, 삭제 정책 |
| FAQ | 민정기 | 인기 매뉴얼/워키 조회, 메인 요약 |
| 알림 | 민정기 | 이벤트 알림, SSE/폴링 |
| Elasticsearch | 민정기 | 벡터 스토어, 매뉴얼/워키 청크 인덱싱, kNN 검색 |
| 티켓 | 김진혁 | 생성, 상태 전이, 라우팅, 이관, 공통 접수 큐 |
| 지식화 | 김진혁 + 김가영 | 지식화 후보 등록(김진혁), TEAM_ADMIN 검수(김가영) |
| 관리자 대시보드 | 김가영 | TEAM_ADMIN/SYSTEM_ADMIN 큐, admin_logs, 운영 지표 |
| 포인트/ESG | 김가영 | 포인트 이력, ESG 등급, ESG 지표 카드 |
| 프론트엔드 | 황희수 | 전체 화면, API 연동, 발표 시연 UX |

## 발표용 MVP 시연 범위

반드시 시연할 것:
1. 회원가입 / 로그인
2. 챗봇 질문 → 매뉴얼/워키 출처 포함 응답
3. 챗봇 실패 → 요청 티켓 발행
4. 티켓 자동 배정 / 공통 접수 큐 분기
5. TEAM_ADMIN 팀원 배정 + 이관 요청
6. 담당자 처리 완료 → 지식화 후보 등록/검수/워키 반영
7. 실시간 알림 확인
8. 포인트 적립 / ESG 등급 산정
9. ESG 지표 카드
10. 관리자 대시보드
11. 배포 URL에서 프론트 화면으로 전체 시연
